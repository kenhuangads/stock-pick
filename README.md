# ⚡ 台股當沖選股 DayTrade Picker

台股當沖交易的**智能選股 → 每日復盤 → 迭代優化**一條龍工具。每個交易日收盤後全自動更新，跨裝置連網即用，手機體驗優先。

**🌐 線上使用：<https://kenhuangads.github.io/stock-pick/>**

> ⚠️ 本工具為程式化研究與教育用途，所有數據與模擬結果僅供參考，**非投資建議**。當沖交易風險極高，違約交割後果嚴重，請自行評估並承擔交易風險。

## 功能總覽

| 功能 | 說明 |
| --- | --- |
| 🎯 **今日選股** | 每日收盤後以 8 大指標掃描上市＋上櫃全市場，依「策略權重加總」排序推薦，並附 CDP 建議買進／停利／停損價 |
| 🔍 **自訂選股** | 自訂價格帶、N 日均量張數、月均振幅、當沖比率、策略複選等條件，在瀏覽器端即時重新選股（例：近 3 日均量 ≥ 5,000 張且股價 100–150 元） |
| 📒 **每日復盤** | 自動用隔日實際行情驗證前一日建議單，計算含手續費（折數可自訂，預設 2.8 折）與當沖證交稅的淨損益，累積損益曲線圖表化 |
| 🧬 **迭代優化** | 滾動 20 個復盤日統計各策略勝率／期望值，**期望值轉負自動停用（汰弱）、績效好加權（留強）**，並回饋到隔日選股排序，全程留有調整紀錄 |
| 📐 **價格模型迭代** | 進出場建議價與**出場引擎**（地板式移動停利、12:00 時間停損）都進迭代迴圈：每日重放最近 20 個復盤日 A/B 實證，目標「成交率 ≥ 67% 硬門檻 → 正期望值中優先賺賠比 ≥ 1.2 → 淨損益最高」（大賺小賠），績效明顯改善才切換 |
| 🧪 **候選策略池** | 新策略以**權重 0 虛擬追蹤**（觸發照記、不影響選股），樣本足夠且期望值實證轉正**自動納入計分**，轉負退回觀察區——策略庫可持續擴充而不冒風險。含技術面與**籌碼面**（三大法人買賣超、融資融券）候選 |
| 🛡️ **每日虧損風控** | 部位風險預算＋每日虧損斷路器：復盤即時試算「把單日綜合虧損壓在上限內」（預設 $10,000）的損益。**進場前風險閘門**在數學上保證單日淨虧損不超過上限；上限／每筆風險／連虧停手筆數皆可在設定即時調整 |
| 🌤 **大盤環境濾網** | 市場寬度（上漲家數比）5 日均 < 50% 的**空方環境當晚不出建議單**——A/B 實證：與動能濾網疊加後單筆期望值 3 倍、回撤大減；「空手也是一種部位」 |
| 📊 **實證儀表板** | 復盤頁一眼掌握：賺賠比／獲利因子／每筆期望值／最大回撤／最慘單日／出場原因分佈條（錢從哪賺、往哪虧）；選股卡附 20 日走勢 sparkline 與大盤環境徽章 |
| 📱 **跨裝置** | 純靜態網頁（GitHub Pages），手機／平板／電腦開瀏覽器即用；支援加入主畫面（PWA） |

## 智能選股：使用哪些指標？

### 基礎濾網（先淘汰不合格標的）

| 濾網 | 預設門檻 | 依據 |
| --- | --- | --- |
| 價格帶 | 10–1000 元 | 排除雞蛋水餃股與超高價股 |
| 流動性 | 近 5 日均成交值 ≥ 5,000 萬、均量 ≥ 2,000 張 | 大單進出不滑價 |
| 波動度 | 近 30 日平均振幅 ≥ 2.5% | 扣除 0.435% 摩擦成本後仍有價差空間 |
| **動能下限** | 訊號日漲幅 ≥ 5% | 70 日 walk-forward 實證：漲幅 <5% 的建議單平均為負期望值（美股 stocks-in-play 理論同理——當沖要挑「事件級動能」標的） |
| 合規性 | 排除**處置股**、**注意股**、非現股當沖標的 | 處置股撮合改分盤、禁現沖，是當沖絕對禁區 |

### 計分策略（複選觸發、權重加總 = 綜合分數）

| # | 策略 | 觸發條件 | 邏輯 |
| --- | --- | --- | --- |
| 1 | 量能突增 | 今日量 ≥ 前 5 日均量 1.5 倍且收紅 | 新資金進場，隔日延續機率高 |
| 2 | 均線多頭 | 5MA > 10MA > 20MA 且收盤 > 5MA | 順大勢做多的結構基礎 |
| 3 | 突破 20 日高 | 收盤創前 20 日新高 | 動能突破，吸引隔日追價 |
| 4 | 當沖熱度 | 當沖比率 ≥ 40% 或較 5 日均值 +15pp | 極短線熱錢湧入（上市個股） |
| 5 | 跳動甜蜜點 | 收盤價落在 100–130／500–650／1000–1300 元 | 剛跨越 tick 門檻，1–2 檔即回本、資金效率最高 |
| 6 | 高波動體質 | 近 30 日均振幅 ≥ 3.5% | 當沖利潤來源 |
| 7 | 順勢拉回 5MA | 多頭排列下回測 5MA 不破 | 「順大勢、逆小勢」低風險買點 |
| 8 | 高週轉率 | 單日週轉率 ≥ 5% | 籌碼換手積極 |

### 候選策略池（虛擬追蹤 → 實證轉正）

| # | 候選策略 | 觸發條件 | 邏輯 |
| --- | --- | --- | --- |
| 9 | 長紅強勢 | 漲幅 ≥ 2%、紅K實體 ≥ 60% 且收在區間頂部 20% | 尾盤買方仍強，隔日慣性延續 |
| 10 | 跳空缺口不補 | 開盤跳空過昨高、全日不回補缺口且收紅 | 多方力道明確的強勢型態 |
| 11 | 布林上軌突破 | 收盤 > 20MA + 2σ | 統計性波動擴張突破 |
| 12 | 三日量價齊揚 | 連 3 日收漲且今日量能不縮 | 趨勢連續性 + 資金持續進場 |
| 13 | 法人買超（籌碼） | 三大法人合計淨買超 ≥ 500 張且收紅 | 法人資金認同、隔日慣性偏多 |
| 14 | 軋空題材（籌碼） | 券資比 ≥ 15% 且融券較前日增加、收紅 | 空方回補潛在燃料 |
| 15 | 低回本檔數 | 含稅費回本 ≤ 2 個跳動檔 | 跳動甜蜜點的直接量化，資金效率最高、暴露風險時間最短 |

> 📊 **關於基本面／籌碼面**：當沖是日內遊戲，基本面（EPS／本益比）對隔日盤中幾乎無預測力，故僅以流動性／波動度（＝當沖語境的「基本面」）作基礎濾網。籌碼面則以候選策略導入——**是否真能提高勝率不用猜，由復盤數據實證**（三大法人、融資融券為官方免費資料；價值最高的「隔日沖分點偵測」因官方無免費 API 未納入）。

候選策略**預設權重 0**：觸發會記進復盤累積統計，但不影響選股；窗口內樣本 ≥ 8 筆且期望值轉正才自動啟用（「實證轉正」），轉負則退回觀察區。**要實驗新想法，只需在 `scripts/strategies.py` 的 STRATEGIES 加一條 `candidate: True` 定義**，其餘交給迭代機制自動驗證。

**權重不是固定的**：每天由復盤結果滾動調整（見下方「迭代優化」），表現差的策略會被自動停用。

### 建議價位（CDP 逆勢系統 + 價格模型迭代）

以最新一日 H／L／C 計算：`CDP=(H+L+2C)/4`、`AH=CDP+(H−L)`、`NH=2CDP−L`、`NL=2CDP−H`、`AL=CDP−(H−L)`，並對齊台股合法 tick。

- **買進**掛 NL、**停利** NH、**停損** AL——三者再各加上「價格模型偏移 × 當日振幅 R」
- 偏移每日迭代（走 walk-forward、不偷看未來）：以最近 20 個復盤日重放搜尋，**成交率 ≥ 67% 為第一硬門檻**
  （target_fill_rate 可調）；達標組合中依目標函數挑選——預設 `objective=payoff`：**正期望值組合中優先
  「賺賠比 ≥ 1.2」者取淨損益最高**（直接編碼「大賺小賠＋高期望值」）；無組合達標時取成交率最高者逼近目標；
  需明顯改善才切換（防雜訊抖動）
- **出場引擎也在搜尋空間內**（每日 A/B 實證自動開關）：
  - **地板式移動停利**（trail，預設候選 0.3R）：觸及停利價後不立刻出場，改追蹤「高點 − 0.3R」回落出場、
    原停利價為地板——實測 83% 的停利單觸價後仍續漲（中位 +0.19R），讓大賺的尾巴多跑
  - **12:00 時間停損**（tstop）：中午前未觸停利即出場——實測拖到收盤沖銷的單午後平均負漂移
    （研究報告的「時間停損法」）
  - 兩者僅作用於 5 分K可核實的交易；日K fallback 維持保守規則
- AH（最高值）為順勢突破加碼參考
- 卡片同時標示「回本約 N 檔」＝該價位含稅費的損益兩平跳動檔數
- 「每日復盤」頁提供診斷：掛單成交率、停利／停損出場占比、**掛價過低錯失率**（未成交且行情跑掉），與全部調整軌跡

## 每日復盤：模擬規則（5 分K 順序核實）

1. 隔日以建議價掛**限價買單**，**以 5 分K逐根還原日內順序**成交：開盤 ≤ 掛價以開盤價成交；
   盤中則需**穿價**（K 棒最低 **<** 掛價）才算成交——「恰好觸價」視為排隊買不到
   （70 日實測 11% 的觸價屬此類，真實市場有排隊風險，寧可少算不多算）
2. 成交那一根**不認停利**——同根內的高點可能發生在買進之前，吃不到就是吃不到；但認停損（保守）
3. 成交之後逐根檢查：先停損（低 ≤ AL）→ 再停利（高 ≥ NH，若移動停利開啟則轉為追蹤高點回落、原停利為地板）
   → 時間停損（開啟時 12:00 前未觸停利即出場）→ 收盤前皆未觸及 → 收盤價沖銷
4. **跳空穿越修正**：開盤（或成交當根）已低於停損價 → 以實際可成交價出場，不得以高於市場的停損價成交
   （停損單觸發後是市價單，跳空時就是吃到跳空價——模擬不許憑空獲利）
5. **跌停鎖死修正**：停損觸發的那根 K 若已觸及跌停價，出場價以跌停價計——跌停鎖死時賣單堆積，
   成交不可能優於跌停（尾部風險不許低估）
6. 為什麼需要 5 分K：日K 只有 OHLC、看不出先後——「先衝高觸停利、午後才回落到掛買價」的單，
   日K模擬會誤記成吃得到的停利（順序偏誤、虛增績效）；5 分K 徹底修正這件事
7. 5 分K 取自 Yahoo Finance（上市 .TW／上櫃 .TWO，約可回溯 60 天），存於 `data/intraday/`；
   無資料的個股 fallback 日K悲觀規則並於紀錄標記 `sim_mode`
8. 費用：手續費 `成交金額 × 0.1425% × 折數`（買賣各一次、低於最低手續費以最低計）＋當沖證交稅 `賣出金額 × 0.15%`
9. 前端可隨時修改**手續費折數／最低手續費／模擬張數**，歷史損益即時重算

> 💡 引入 5 分K核實後，模擬績效比日K版「難看但誠實」——這正是復盤的意義：
> 在模擬裡發現吃不到的獲利，好過在真金白銀裡發現。

### 這些規則怎麼來的？（walk-forward A/B 實證）

所有濾網與假設都經過「同一段歷史、只變一個條件」的 walk-forward 重放對照（無前視偏誤），擇優落地：

| 變體（70 個復盤日） | 出手日 | 勝率 | 累計淨損益 | 單筆期望值 | 賺賠比 | 最大回撤 |
| --- | --- | --- | --- | --- | --- | --- |
| 寬鬆成交假設（觸價即成交） | 70 | 47.6% | +72,620 | +183 | 1.18 | −115,402 |
| 誠實成交（穿價＋跌停修正） | 70 | 42.8% | −188,250 | −492 | 1.10 | −258,654 |
| 誠實＋動能 ≥5% | 70 | 45.3% | +108,495 | +286 | 1.35 | −213,173 |
| **誠實＋動能＋環境濾網（採用）** | **35** | **43.0%** | **+151,824** | **+883** | **1.87** | **−87,152** |

- 「觸價即成交」的獲利有一大塊是**限價單逆選擇**的幻覺：輕觸掛價就反彈的多是贏家（真實會排隊買不到），穿價成交的偏向續跌的輸家——誠實假設一開，基準立刻轉虧，這就是要誠實的原因
- 動能與環境兩個濾網**單獨用都不夠、疊加有強正交互**：只在多方環境出手＋只挑強動能標的，出手日減半、單筆期望值 3 倍、回撤縮 2/3
- 更緊的停損網格／更早的時間停損選項實測**無增益**，不採用（避免過擬合）

## 每日虧損風控（部位風險預算＋斷路器）

當沖穩定的核心是「截斷虧損」。工具在復盤即時試算風控後的損益（前端可調，不改變選股）：

1. **部位風險預算**：每筆依「進場−停損」距離反推張數，使每筆最大風險約等於 `per_trade_risk`（預設 $2,500），夾在 1～`max_lots` 張——寬停損的高價股自動縮小部位
2. **進場前風險閘門**：依綜合分數（信心）順序進場，只有當「這筆觸停損的最大虧損 ≤ 當日剩餘額度」才進場。
   一般情況下單日淨虧損不超過上限（預設 $10,000）；惟**跳空穿越停損**時實際出場價低於停損價，
   單日虧損仍可能小幅超限（誠實模擬不迴避此尾部風險）
3. **連虧停手**：當日連虧達 `max_consecutive_losses` 筆（預設 3，即研究報告的鐵律）即停開新倉
4. 復盤頁同時呈現「風控後」與「未風控」累計損益對照、觸發斷路器天數、單日虧損破上限天數（實測種子資料：**原始 8 天 → 風控後 0 天**）
5. **每日上限敏感度分析**：並排不同上限（$1 萬～無上限）下的累計損益、勝率、最慘單日、最大回撤與「報酬÷最大回撤」（Calmar 式風險調整報酬），標出有上限組合中的最佳並可一鍵套用。刻意**不自動最佳化上限本身**——純看報酬幾乎必然「越高越賺」，小樣本下少數尾部大賺還會讓「無上限」連風險調整比都偏高，故上限維持你的風險偏好、由你拍板，此表只幫你看清取捨

> 🎯 三個終極目標的務實評估：**掛單成交率 ≥ 67%**（價格模型第一目標，可達）、**每日綜合虧損 < $10,000**（風險閘門保證，可達）、**勝率 ≥ 75%**（當沖現實極難，報告言「往往五五開」；系統改追「正期望值下盡量拉高勝率」，55～62% 配正期望值即為優秀當沖系統）。

### 汰弱留強的保底機制與遞補

期望值轉負的策略會被停用；但若同時停用太多（啟用 < `min_active`，預設 6），系統會選不出股票、
停止累積新樣本而永遠癱瘓（死亡螺旋）。此時自動把「停用中期望值最佳」的策略以最低權重保底復活，
維持系統持續選股、持續學習，待窗口績效回升再正常加權。

**每日固定 `max_picks`（預設 8）檔**：弱勢窗口下完整門檻（≥2 個活躍策略命中）選不滿時，
從「通過基礎濾網且有觸發訊號」者依（活躍命中數、綜合分數、總命中數、振幅）遞補湊滿，
卡片標「遞補」明示信心較低；遞補單照常復盤，維持樣本累積讓停用策略有機會實證翻身。

## 迭代優化：汰弱留強怎麼運作？

```
每日收盤後：
  復盤結果 → 統計「最近 20 個復盤日」各策略背書交易的勝率與期望值
    ├─ 核心策略：樣本 ≥ 8 筆且期望值 < 0 → 停用（權重 0，持續虛擬追蹤）；回升自動重新啟用
    ├─ 候選策略：預設權重 0 觀察中；樣本 ≥ 8 筆且期望值轉正 → 實證轉正、自動納入計分
    └─ 其餘依勝率+期望值映射權重（0.3 ~ 2.0）
  價格模型 → 重放窗口搜尋最佳進出場偏移（明顯改善才切換）
  新權重 + 新偏移 → 立即用於當晚產生的隔日推薦排序與建議價
  所有啟停/偏移變化 → 寫入「汰弱留強紀錄」「價格模型紀錄」供檢視
```

```
GitHub Actions（平日晚間，21:10 起多發兜底）
  └─ scripts/daily_update.py
       1. 抓 TWSE/TPEx 收盤行情＋當沖統計＋處置/注意/可當沖名單＋籌碼（三大法人、融資融券）→ data/history/
       2. 復盤前一日建議單 → data/reviews.json
       3. 迭代優化策略權重 → data/latest/strategies.json
       4. 迭代優化價格偏移 → data/latest/price_model.json
       5. 全市場掃描選股   → data/latest/picks.json、market.json
  └─ commit 回 repo → 自動部署 GitHub Pages（純靜態前端讀取上述 JSON）
```

- 資料來源：[臺灣證券交易所 OpenAPI](https://openapi.twse.com.tw/)、[TWSE rwd/exchangeReport](https://www.twse.com.tw/)、[證券櫃檯買賣中心 OpenAPI 與行情端點](https://www.tpex.org.tw/openapi/)；籌碼面用 TWSE `T86`（三大法人）、`MI_MARGN`（融資融券）與 TPEx 對應端點
- 上櫃個股無公開當沖統計，故「當沖熱度」策略僅適用上市個股；上櫃籌碼 openapi 僅提供最新日、歷史無法回補
- `data/history/` 快照與復盤紀錄**永久保留**（`config.history_window: 0`），復盤自 2026-04-01 起；
  Yahoo 5 分K僅回溯約 60 天，故較早期間（約 2026-05 中以前）的復盤為日K保守模式（`sim_mode: daily`），
  含順序偏誤、解讀時應以 5 分K核實子集為準

## 本機開發

```bash
pip install -r requirements.txt
python scripts/backfill.py 45        # 回補 45 個交易日歷史行情（可續跑）
python scripts/backfill_chip.py      # 為既有歷史補三大法人／融資融券（冪等）
python scripts/daily_update.py --rebuild   # walk-forward 重建復盤、策略權重與價格偏移
python scripts/daily_update.py       # 每日更新（等同 Actions 排程做的事）
python -m http.server 8000           # 本機預覽前端
```

調整選股門檻改 `config.json`（濾網、每日推薦檔數、優化器窗口等），前端個人設定（折數、張數、自訂條件）存於瀏覽器 localStorage。

## GitHub Actions

| Workflow | 觸發 | 用途 |
| --- | --- | --- |
| `每日更新` | 平日晚間多發（21:10/21:40/22:20/23:10/00:10 台北）／手動 | 抓資料→復盤→審計→優化→選股→commit（冪等，先成功者更新後其餘快速跳過） |
| `歷史資料回補` | 手動 | 首次建置或補資料缺口，可選 walk-forward 重建 |

Pages 採 **branch 部署**（main 根目錄、含 `.nojekyll`）：任何 push 後 GitHub 自動建置上線，無需部署 workflow。

## 已知限制與模擬假設

- 復盤以 5 分K 還原日內順序（含與官方日K的一致性防線），但 5 分K 內部的先後仍屬未知，同根內採悲觀假設；實際盤中操作結果仍會與模擬不同
- 建議單假設隔日可於掛價成交，未考慮開盤跳空鎖死、盤中流動性瞬間消失等極端情況
- 處置／注意名單於選股當晚生效；盤中新增的異常管制無法即時反映
- 本工具不含下單功能，也不應被用於自動下單
