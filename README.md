# ⚡ 台股當沖選股 DayTrade Picker

台股當沖交易的**智能選股 → 每日復盤 → 迭代優化**一條龍工具。每個交易日收盤後全自動更新，跨裝置連網即用，手機體驗優先。

**🌐 線上使用：<https://kenhuangads.github.io/stock-pick/>**

> ⚠️ 本工具為程式化研究與教育用途，所有數據與模擬結果僅供參考，**非投資建議**。當沖交易風險極高，違約交割後果嚴重，請自行評估並承擔交易風險。

---

## 功能總覽

| 功能 | 說明 |
| --- | --- |
| 🎯 **今日選股** | 每日收盤後以 8 大指標掃描上市＋上櫃全市場，依「策略權重加總」排序推薦，並附 CDP 建議買進／停利／停損價 |
| 🔍 **自訂選股** | 自訂價格帶、N 日均量張數、月均振幅、當沖比率、策略複選等條件，在瀏覽器端即時重新選股（例：近 3 日均量 ≥ 5,000 張且股價 100–150 元） |
| 📒 **每日復盤** | 自動用隔日實際行情驗證前一日建議單，計算含手續費（折數可自訂，預設 2.8 折）與當沖證交稅的淨損益，累積損益曲線圖表化 |
| 🧬 **迭代優化** | 滾動 20 個復盤日統計各策略勝率／期望值，**期望值轉負自動停用（汰弱）、績效好加權（留強）**，並回饋到隔日選股排序，全程留有調整紀錄 |
| 📱 **跨裝置** | 純靜態網頁（GitHub Pages），手機／平板／電腦開瀏覽器即用；支援加入主畫面（PWA） |

## 智能選股：使用哪些指標？

### 基礎濾網（先淘汰不合格標的）

| 濾網 | 預設門檻 | 依據 |
| --- | --- | --- |
| 價格帶 | 10–1000 元 | 排除雞蛋水餃股與超高價股 |
| 流動性 | 近 5 日均成交值 ≥ 5,000 萬、均量 ≥ 2,000 張 | 大單進出不滑價 |
| 波動度 | 近 30 日平均振幅 ≥ 2.5% | 扣除 0.435% 摩擦成本後仍有價差空間 |
| 合規性 | 排除**處置股**、**注意股**、非現股當沖標的 | 處置股撮合改分盤、禁現沖，是當沖絕對禁區 |

### 計分策略（複選觸發、權重加總 = 綜合分數）

| # | 策略 | 觸發條件 | 邏輯 |
| --- | --- | --- | --- |
| 1 | 量能突增 | 今日量 ≥ 前 5 日均量 1.5 倍且收紅 | 新資金進場，隔日延續機率高 |
| 2 | 均線多頭 | 5MA > 10MA > 20MA 且收盤 > 5MA | 順大勢做多的結構基礎 |
| 3 | 突破 20 日高 | 收盤創前 20 日新高 | 動能突破，吸引隔日追價 |
| 4 | 當沖熱度 | 當沖比率 ≥ 40% 或較 5 日均值 +15pp | 極短線熱錢湧入（上市個股） |
| 5 | 跳動甜蜜點 | 收盤價落在 100–130／500–650／1000–1300 元 | 剛跨越 tick 門檻，1–2 檔即回本、資金效率最高 |
| 6 | 高波動體質 | 近 30 日均振幅 ≥ 3.5% | 當沖利潤來源 |
| 7 | 順勢拉回 5MA | 多頭排列下回測 5MA 不破 | 「順大勢、逆小勢」低風險買點 |
| 8 | 高週轉率 | 單日週轉率 ≥ 5% | 籌碼換手積極 |

**權重不是固定的**：每天由復盤結果滾動調整（見下方「迭代優化」），表現差的策略會被自動停用。

### 建議價位（CDP 逆勢系統）

以最新一日 H／L／C 計算：`CDP=(H+L+2C)/4`、`AH=CDP+(H−L)`、`NH=2CDP−L`、`NL=2CDP−H`、`AL=CDP−(H−L)`，並對齊台股合法 tick。

- **買進**掛 NL（近低值）、**停利** NH（近高值）、**停損** AL（最低值）
- AH（最高值）為順勢突破加碼參考
- 卡片同時標示「回本約 N 檔」＝該價位含稅費的損益兩平跳動檔數

## 每日復盤：模擬規則（保守原則）

1. 隔日以建議價 NL 掛**限價買單**：當日最低價 ≤ NL 才視為成交；開盤低於 NL 則以開盤價成交
2. 出場判斷採**悲觀排序**（OHLC 無法得知日內先後）：先檢查停損（最低 ≤ AL → 以 AL 認賠）→ 再檢查停利（最高 ≥ NH → 以 NH 獲利）→ 皆未觸及 → 收盤價沖銷
3. 費用：手續費 `成交金額 × 0.1425% × 折數`（買賣各一次、低於最低手續費以最低計）＋當沖證交稅 `賣出金額 × 0.15%`
4. 前端可隨時修改**手續費折數／最低手續費／模擬張數**，歷史損益即時重算

## 迭代優化：汰弱留強怎麼運作？

```
每日收盤後：
  復盤結果 → 統計「最近 20 個復盤日」各策略背書交易的勝率與期望值
    ├─ 樣本 ≥ 8 筆且期望值 < 0  → 停用該策略（權重 0，持續虛擬追蹤）
    ├─ 表現回升                → 自動重新啟用
    └─ 其餘依勝率+期望值映射權重（0.3 ~ 2.0）
  新權重 → 立即用於當晚產生的隔日推薦排序
  所有啟停變化 → 寫入「汰弱留強紀錄」供檢視
```

## 系統架構

```
GitHub Actions（平日 21:00 台北時間）
  └─ scripts/daily_update.py
       1. 抓 TWSE/TPEx 收盤行情＋個股當沖統計＋處置/注意/可當沖名單 → data/history/
       2. 復盤前一日建議單 → data/reviews.json
       3. 迭代優化策略權重 → data/latest/strategies.json
       4. 全市場掃描選股   → data/latest/picks.json、market.json
  └─ commit 回 repo → 自動部署 GitHub Pages（純靜態前端讀取上述 JSON）
```

- 資料來源：[臺灣證券交易所 OpenAPI](https://openapi.twse.com.tw/)、[TWSE rwd/exchangeReport](https://www.twse.com.tw/)、[證券櫃檯買賣中心 OpenAPI 與行情端點](https://www.tpex.org.tw/openapi/)
- 上櫃個股無公開當沖統計，故「當沖熱度」策略僅適用上市個股
- `data/history/` 僅保留最近 60 個交易日快照（復盤紀錄永久保留）

## 本機開發

```bash
pip install -r requirements.txt
python scripts/backfill.py 45        # 回補 45 個交易日歷史（可續跑）
python scripts/daily_update.py --rebuild   # walk-forward 重建復盤與策略權重
python scripts/daily_update.py       # 每日更新（等同 Actions 排程做的事）
python -m http.server 8000           # 本機預覽前端
```

調整選股門檻改 `config.json`（濾網、每日推薦檔數、優化器窗口等），前端個人設定（折數、張數、自訂條件）存於瀏覽器 localStorage。

## GitHub Actions

| Workflow | 觸發 | 用途 |
| --- | --- | --- |
| `每日更新` | 平日 21:00（台北）／手動 | 抓資料→復盤→優化→選股→commit→部署 |
| `歷史資料回補` | 手動 | 首次建置或補資料缺口，可選 walk-forward 重建 |

## 已知限制與模擬假設

- 復盤以日 K 的 OHLC 模擬，無法還原日內先後順序，故採「停損優先」的最壞情境假設；實際盤中操作結果會與模擬不同
- 建議單假設隔日可於掛價成交，未考慮開盤跳空鎖死、盤中流動性瞬間消失等極端情況
- 處置／注意名單於選股當晚生效；盤中新增的異常管制無法即時反映
- 本工具不含下單功能，也不應被用於自動下單
